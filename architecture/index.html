Bagvedliggende løsningsflow

Voice, transskription og data samles før Mikkel får svaret

Telefonnummeret identificerer kunden. Den transskriberede samtale identificerer spørgsmålet. SIGNAL AGENT orkestrerer derefter opslag, forklaring og sagsnotat med human approval.

01 Contact center

Indgående opkald

Eksisterende telefoni viser nummer, kø, ventetid og aktiv agent. Samtalen optages som i normal telco-drift.

Telefonnummer Call-ID Agent-ID
02 Voice layer

Live transcript

Voice agenten sender løbende transcript og nøgleord til SIGNAL AGENT, mens Mikkel stadig taler med kunden.

“Regning højere” Intent: faktura Entity: maj-faktura
03 SIGNAL AGENT

Orkestrering

SIGNAL AGENT kombinerer transcript, kunde-match og datakilder i én kontrolleret intern agent.

Reasoning Find sandsynlig årsag til afvigelsen.
Policy check Kontroller svar mod produktregler og samtykke.
Audit Log kilder, tid og Mikkels endelige handling.
04 Datakilder

Opslag på tværs

Agenten henter kun de kilder, der er relevante for den identificerede journey.

CRM og kundeprofil Faktura og betaling Forbrug og tilvalg Produktregler og historik
05 Agent workspace

Mikkel godkender

SIGNAL AGENT viser årsag, muligt svar, kilder og sagsnotat. Kunden kontaktes først, når Mikkel godkender.

Forklaring Svarforslag Sagsnotat
Realistisk integration

Løsningen forudsætter ikke en kundevendt chatbot. Den kobles på eksisterende contact center, call recording/transcription og interne system-API'er eller RPA-connectors.

Governance by design

SIGNAL AGENT sender ikke automatisk svar til kunden. Medarbejderen godkender, og alle kilder gemmes i audit-log og CRM.
